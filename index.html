Git Status

git git status

Muestra el estado del working copy. Hay 3 estados: Staged o listo para commitear. Unstaged, cambiado pero que no entra en el proximo commit. y Untracked: No esta siendo trackeado por git.

Svn Checkout

svn svn checkout svn+ssh://server/subversion/repo

git git svn clone -r1400:HEAD -s svn+ssh://server/subversion/repo

Explicacion: Se esta clonando ( analogo a checkout en svn) un repositorio, desde la revision 1400 hasta la ultima ( HEAD ), con un layout estandard (-s, osea trunk, tags, branches) y la url del repositorio base.

Svn add

svn svn add

git git add

Explicacion: Se agregan archivos untracked o unstaged a stage.

Svn Commit

svn svn commit -a -m "mensaje"

git git commit -a -m "mensaje"

git git svn dcommit

Explicacion: Se esta generando un commit con los archivos modificados y con un mensaje "mensaje". Luego se esta "pusheando"

Svn Update

svn svn update

git git svn rebase

Explicacion: Se actualiza el branch local, con el branch del remoto(svn en este caso)

Svn Revert

svn svn revert

git git checkout -- <file>

Explicacion: Se "checkoutea" los archivos a su estado sin modificar. Esto saca a los archivos del estado "unstaged".

Svn Log

svn svn log

git git log <file>

Explicacion: Sin muchas vueltas, retorna un log reverso de commits en le branch actual. NO es remoto, es local. Mucho mas rapido, los issues que no estan en el remoto, se identifican porque no se ve el git-svn-id. Toma argumento opcional de archivo, que permite ver el history de cambios de ese archivo

Stash

git git stash && git stash apply

Git stash, levanta los cambos unstaged y staged. Los guarda localmente para su posterior uso. Se vuelven a aplicar con git stash apply. Sumamente util para switchear lo que uno esta haciendo sin perder cambios o tenes commits desordenados.
